About DishBoard

Dishboard is a recipe website where a user can upload their own recipe or can search for a recipe for his or hers own use.

On this page we go in detail about all the entities that are used in this project and all the features/pages that are going to be possibile in the web application.

User

A user makes an account by entering his or hers first name, last name, email, password and date of birth.
After this a user can write about him/her self to give a bit more personal flair to the profile.
A user can make a recipe on the website and it will be automatically uploaded on the website.

Recipe

If a user wants to make a recipe they need to give it the following properties:
- Name of the recipes,
- Time in minutes,
- For how many people the recipe is indicated by a number,
- List of ingredients (see ingredients how an ingredient is build),
- Total Nutrition calculated by the different ingredients nutrition,
- cooking instructions on how to prepare and make the meal,
- Optionally a user can give tips.
- An image url to have a nice image for use.

A recipe needs to have atleast 2 ingredients to make it. Cooking instructions is an array of descriptions which needs to have a minimum of 1 cooking step
Total nutriton of each nutrition value cannot be calculated if one or more ingredients does not have the nutrition value filled in by the user.

Ingredient

An ingredient has name, Amount in numbers ,Size determined by an enum value, Nutrition value (see nutrition value on how it is made).
If you want to fill in a value you need to select a type of weight and add the appropiate amount of it for the recipe.

Nutrition Value

In order to fill in the total nutritional value for an ingredient a user needs to fill in the following properties:
-Calories
-Carbohydrates
-Sodium
-Protein
-Fat
-Fiber
A user does not need to fill in all the nutritional values because sometimes the data is not available for the user. However this means the recipe does not calculate the total nutritional values.

HTTP Route Output
GET /user/ List of users
GET /user/:id Specific user
GET /user/following/:id Get list of users that the user follows
POST /user/register User object and a token, places the user in the DB
POST /user/login User object and a token, logs the user in the website
POST /user/follow/:id Adds the user ID to the logged in user ID
PUT /user/:id Changes the user data
DELETE /user/:id Deletes the user data
GET /recipe/ Gets all the recipe data
GET /recipe/:id Get the specific recipe data
POST /user/ Posts the recipe data
PUT /user/:id Updates the recipe data
DELETE /recipe/:id Deletes the recipe data
