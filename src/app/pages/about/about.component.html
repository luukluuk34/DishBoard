About DishBoard

Dishboard is a recipe website where a user can upload their own recipe or can search for a recipe for his or hers own use.

On this page we go in detail about all the entities that are used in this project and all the features/pages that are going to be possibile in the web application.

User

A user makes an account by entering his or hers first name, last name, email and date of birth.
After this a user can write about him/her self to give a bit more personal flair to the profile.
A user can make a recipe on the website and it will be automatically added to their profile and uploaded on the website.
A user can also save recipes that he or she likes so it is easily available for use.

Recipe

If a user wants to make a recipe they need to give it the following properties:
- Name of the recipes,
- Time in minutes,
- For how many people the recipe is indicated by a number,
- List of ingredients (see ingredients how an ingredient is build),
- Total Nutrition calculated by the different ingredients nutrition,
- Description on how to prepare and make the meal,
- Optionally a user can give tips.

A recipe needs to have atleast 3 ingredients to make it.
Total nutriton of each nutrition value cannot be calculated if one or more ingredients does not have the nutrition value filled in by the user.

Ingredient

An ingredient has name, amount determined by an enum value, Nutrition value (see nutrition value on how it is made).
If you want to fill in a value you need to select a type of weight and add the appropiate amount of it for the recipe.

Nutrition Value

In order to fill in the total nutritional value for an ingredient a user needs to fill in the following properties:
-Calories
-Carbohydrates
-Sodium
-Protein
-Fat
-Fiber
A user does not need to fill in all the nutritional values because sometimes the data is not available for the user. However this means the recipe does not calculate the total nutritional values.
